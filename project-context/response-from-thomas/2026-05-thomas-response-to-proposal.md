## Why he's removing each piece

### Phase 1 — Foundation & Tooling (REMOVE as a separately-scheduled phase)
Reason: Meta Pixel, Google Search Console, and Google Business Profile are already live. Current site is small enough that there isn't a real audit to run. GA4 custom events + sitemap resubmission have to happen *after* the rebuild anyway because URLs all change. → Fold the small post-build tracking work into the main build.

### Phase 3 — SEO + AI Optimization (REMOVE as a separately-scheduled phase)
Reason: He sees schema, RankMath config, `llms.txt`, and Core Web Vitals as **part of building the site properly**, not a separate two-week phase. Not looking for an ongoing SEO retainer. He'll provision any accounts himself in Cloud Title's name. → Include as standard build deliverable.

### Phase 4 — Auto-Blog Engine + 90-day ops (REMOVE)
Reason: Cloud Title **already produces 20–30 pieces of content per day** across their channels through their own AI stack. The WordPress API is there for them to publish into. So the auto-blog engine isn't added value for them. → Keep the blog *build itself* (which is in Phase 2); drop the engine and the managed posting.

### Phase 5 — Inside-Sales Follow-Up (REMOVE)
Reason: He liked the GHL Contact 360 work but is **building this internally**. Long-cycle relationship work (5–10 files per agent over time) is core to his own build. Door open for our input later, but doesn't want this engagement scoped around setting up its data layer.
